JSX 的本质

JSX 元素节点会被 babel 编译成 React Element 形式。
React.createElement 的用法：

createElement参数：

  • 第一个参数： 如果时组件类型，会传入组件对应的类或者函数；如果时 dom 元素，传入 div 或者 p 这种元素标签名字的字符串
  • 第二个参数：一个对象，在 dom 类型中为标签属性，在组件类型中为 props。
  • 其他参数：依次为 children ，根据顺序排列

JSX 的转换规则

jsx元素类型 react.createElement 转换后 type 属性
element元素类型 react element类型 标签字符串，例如 div
fragment类型 react element类型 symbolreact.fragment类型
文本类型 直接字符串 无
数组类型 返回数组结构，里面元素被react.createElement转换 无
组件类型 react element类型 组件类或者组件函数本身

JSX 经过 React 底层调和处理后

最终，在调和阶段，上述 React element 对象的每一个子节点都会形成一个与之对应的 fiber 对象，然后通过 sibling、return、child 将每一个 fiber 对象联系起来。
React 常用的 fiber 类型，以及 element 对象和 fiber 类型的对应关系。

demo：

JSX 调和之后形成的 fiber 结构图

fiber 对应关系：

  • child： 一个由父级 fiber 指向子级 fiber 的指针。
  • return：一个子级 fiber 指向父级 fiber 的指针。
  • sibling: 一个 fiber 指向下一个兄弟 fiber 的指针

注意：

  • 对于上述在 jsx 中写的 map 数组结构的子节点，外层会被加上 fragment ；
  • map 返回数组结构，作为 fragment 的子节点。
